# 🔄 Kodi Sync Dashboard - Complete Setup Guide

## Overview

Your system now has a **complete end-to-end workflow** for discovering, organizing, and managing media links:

```
┌──────────────┐      ┌──────────────┐      ┌──────────────────┐
│ Kodi Addon   │ ---> │ Backend API  │ ---> │ Web Dashboard    │
│ (Discovery)  │      │ (Storage)    │      │ (Organization)   │
└──────────────┘      └──────────────┘      └──────────────────┘
```

---

## 🎯 What It Does

### 1. **Kodi Addon (Media Discovery)**
- Scans all your Kodi addons, configs, and files
- Finds media links (M3U, streams, playlists)
- Archives (ZIP/RAR/7z), SQLite databases, APK files
- Sends everything to the backend API

### 2. **Backend API (Data Storage)**
- Receives links from Kodi addon
- Stores them with metadata (source, type, category)
- Provides REST API endpoints for querying
- Auto-classifies content (Movies/TV/Live TV)

### 3. **Web Dashboard (Organization & Playlists)**
- Fetches all discovered links from backend
- Allows manual classification override
- Validates which URLs are working
- **Generates organized M3U playlists** by category:
  - 🎬 Movies
  - 📺 TV Shows
  - 📡 Live TV
  - 🎥 VOD

---

## 🚀 How to Use

### Step 1: Run a Scan in Kodi

1. **Open Kodi**
2. **Press M key** (or Ctrl+M) to open the addon menu
3. **Select "Scan All Addons & Configs"** (or any scan option)
4. **Wait for completion** - Progress dialog shows real-time progress
5. **Note the Session ID** displayed in the completion message

Example completion message:
```
All-addons scan complete. Files scanned: 1,247. 
URLs found: 523. Batches sent: 11. 
Session ID: addon-abc123-xyz789 
(Use this ID in the web dashboard to view results)
```

⚠️ **IMPORTANT**: Copy or remember your **Session ID** - you'll need it for the web dashboard!

---

### Step 2: Open the Web Dashboard

1. **Open your browser** to: http://localhost:5001
2. **Scroll down** to find the **"🔄 Kodi Sync Dashboard"** section
3. **Enter your Session ID** from Step 1
4. **Click "Load Session"**

---

### Step 3: View & Organize Your Links

Once loaded, you'll see:

#### 📊 **Session Statistics**
- Total jobs
- Completed/Scraping/Queued/Failed counts

#### 🔗 **Discovered URLs**
- All media links found by Kodi
- Auto-classified by content type
- Filter by: All, Movies, TV Shows, Live TV

#### ✅ **Validation**
1. Click **"Validate URLs"** to test which links work
2. Green checkmark ✅ = Working
3. Red X ❌ = Broken

#### 📋 **Generate Playlists**
1. Click **"Generate M3U Playlists"**
2. Three separate playlists are created:
   - **Movies.m3u** - All movie links
   - **TV Series.m3u** - All TV show links
   - **Live TV.m3u** - All live TV links
3. Click **"Download"** on any playlist
4. Use in: VLC, Kodi, IPTV Smarters, TiviMate, Perfect Player, etc.

## 🔧 Technical Details

### Backend API Endpoints
```
GET  /api/kodi-sync/session/:sessionId     - Get all jobs for a session
GET  /api/kodi-sync/status/:jobId          - Get single job status
GET  /api/kodi-sync/results/:jobId         - Get job results (JSON/M3U/CSV)
POST /api/kodi-sync/batch                  - Queue multiple URLs
GET  /api/kodi-sync/health                 - Check API health
```

### Session ID Format
- Auto-generated by Kodi addon
- Format: `addon-{random}-{timestamp}`
- Unique per addon installation
- Persistent across scans

### Content Classification
The system uses these types:
- **movie** - Feature films, documentaries
- **tv-series** - TV shows, episodes
- **live-tv** - Live streaming channels
- **vod** - Video on demand content
- **unknown** - Unclassified content

## 🐛 Troubleshooting

### "No jobs found"
- Check your Session ID is correct
- Make sure backend is running: http://localhost:3001/api/health
- Try running a new scan in Kodi

### "Failed to fetch session"
- Ensure backend is running on port 3001
- Check firewall isn't blocking localhost
- Restart backend if needed

### "No URLs loaded"
- Wait for scan jobs to complete in Kodi
- Completed jobs show green checkmark ✅
- Refresh the session after scans finish

### Backend Not Running
```bash
# Navigate to backend folder
cd backend

# Start the backend
npm start
```

### Frontend Not Running
```bash
# Navigate to project root
cd C:\Users\josht\Downloads\media-link-extractor-main\media-link-extractor-main

# Start the frontend
npm run dev
```
